Publisher — Proposed Architecture

① Material② Guardrails / Persona ③ Checkpoints④ Observability & Alarms
A research concept → a unique, persona-voiced single-page webpage. The agent writes; four separate pillars make constraint-handling invisible to it.
ARCHITECTUREcore problem · components & how they connect · key decisions & rationale · known tradeoffs

Core problem

  • Publishing your work is two hard jobs: sound like you + look good and be well-researched.
  • Generic AI is voiceless; an unconstrained agent can't be trusted to publish.
  • Publisher is the cage that makes a creative agent safe to ship a persona-voiced webpage from a concept.

Components & how they connect

  • Orchestrator (thin loop) — only thing the agent talks to; sequences the 4 pillars.
  • Persona store feeds Guardrails + agent context.
  • Material in → research→build→refine → checkpoints → publish; alarms meter throughout; escalate on critical.

Key decisions & why

  • Persona = declared guardrail, not a hidden prompt (inspectable, enforced twice).
  • Orchestrator mediates — agent never calls pillars (keeps constraints invisible, swap stays clean).
  • Vercel AI SDK → one-line worker swap = the portability bonus.

Known tradeoffs

  • LLM-as-judge for voice/quality — non-deterministic; threshold + escalate, not silent block.
  • Partial portability — core loop portable; web-search + caching are Anthropic-specific.
  • 24h scope — protect the spine; 3rd-party API + cost/latency on the critical path.
DESIGN RATIONALEedge-case handling · key choices · resilience to change · failure modes · alternatives considered

Edge-case handling

  • Concept is empty/thin?
    INPUT_EMPTY → halt; never run the agent on nothing.
  • Not enough research?
    INSUFFICIENT_RESEARCH → loop back to research, then escalate.
  • All gates pass but page is weak?
    High-stakes runs keep a human final-approval; gates aren't claimed infallible.

Key choices · X over Y

  • Persona vs a system prompt?
    Declared, inspectable, reusable — enforced twice & measured by checkpoints.
  • AI SDK vs raw Anthropic SDK?
    One-line provider swap; cost: web tools + caching are provider-specific.
  • Own loop vs an agent framework?
    Harness must own the loop → a checkpoint after every round.

Resilience to change

  • New brand / design system?
    Add design tokens to the persona — no engine change.
  • New channel, 280-char cap?
    Add a guardrail + a different Sink adapter; engine untouched.
  • Tighter latency SLA?
    Add a latency budget → breach fires HIGH_LATENCY.

Failure modes

  • Judge LLM is down?
    Fail-closed — treat as fail + escalate; never fail-open.
  • Refine loop oscillates?
    MAX_ATTEMPTS → WEBPAGE_GENERATION_FAILED + escalate.
  • Provider outage / refusal?
    RATE_LIMITED backoff; REFUSAL/PROVIDER_ERROR escalate; resume from last checkpoint.

Alternatives considered

  • One LLM call + long prompt?
    Fails the rubric (pillars not separable, no replay/alarms/swap) and drifts undetected.
  • Agent calls the pillars itself?
    Leaks the cage into the worker; breaks swappability.
  • Managed Agents / Agent SDK?
    They self-drive their own loop — we need to own it.
